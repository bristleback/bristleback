Bristleback.controller.ActionExceptionHandler Class

Defined in: bristle-controller.js:53
Module: controller
Parent Module: Bristleback

Action exception handler class allows to fully control exception handling process. Instance of this class is created for every action, action class, action callback object and also there is one exception handler in action controller object.

Adds default rendering handler to be used when exception occurs. Before using rendering functionality, templating framework implementation should be provided by calling Bristleback.templateController.registerTemplateFramework() method. Rendering handlers ALWAYS break exception processing cycle.

Parameters:

  • templateName Object

    template name that should be used.

  • containerId Object

    id of the parent container for rendered template.

  • renderingMode String

    name of rendering mode, one of the specified in Bristleback.templateController.renderingModes map. By default, "replace" mode is used.
